Detail Tiap Tahapan
1
Akuisisi Video & Deteksi Pose
Video latihan beban (Squat, Deadlift, Bench Press) diproses frame-per-frame oleh MediaPipe BlazePose, menghasilkan 33 titik landmark 3D per frame. 📄 src/data/extract_pose.py
2
Normalisasi & Pembentukan Tensor
Koordinat dinormalisasi terhadap pusat pinggul (hip-center), kemudian di-resample secara temporal menjadi ukuran tetap 64 frame × 33 sendi × 3 koordinat (x,y,z). 📄 src/data/preprocess.py
3
Auto-Labeling Biomekanis
Validator biomekanis menghitung sudut sendi dengan arccosine trigonometri 3D dan memberikan label otomatis (benar/salah) berdasarkan ambang batas klinis. 📄 src/data/biomechanics_validator.py
4
Training & Inferensi 3D-CNN + Attention
Tensor diumpankan ke model AttentiveSkel3D. Mekanisme atensi biomekanis memperkuat sendi kritis dan menekan noise, lalu lapisan fully-connected mengklasifikasikan gerakan. 📄 src/models/model_3dcnn.py
5
Visualisasi & Interpretasi Atensi
Attention map divisualisasikan pada skeleton: sendi yang melewati threshold 0.4 divisualisasikan merah, menunjukkan fokus model secara interpretatif. 📓 notebooks/06_attention_visualization.ipynb

Landasan Teori
Mengapa representasi skeleton?

Analisis gerakan berbasis video piksel mengandung informasi redundan yang tidak relevan (tekstur pakaian, latar belakang, variasi pencahayaan). Representasi skeleton mereduksi dimensi ini secara drastis sambil mempertahankan informasi kinematika yang sesungguhnya relevan untuk evaluasi gerakan.

"Model berbasis skeleton memiliki keunggulan komputasi karena memfokuskan analisis pada pola pergerakan sendi tanpa dipengaruhi noise pencahayaan, menghasilkan representasi yang jauh lebih ringan." — Noor & Park (2023), ICCVW
Pose Estimation Dimensionality Reduction Invariant Features

Keunggulan Teknis: 33 titik × 3 koordinat = 99 nilai numerik per frame, jauh lebih hemat dibanding resolusi piksel video (contoh: 640×480 = 307.200 nilai per frame).

Tensor akhir berukuran (64 × 33 × 3) yang mewakili 64 frame waktu × 33 sendi × 3 koordinat. Tensor ini yang menjadi input langsung ke 3D-CNN.

📖
Landasan Teori
Mengapa 3D-CNN untuk analisis gerakan?

Konvolusi 2D standar hanya menangkap fitur spasial pada satu frame. Gerakan olahraga adalah fenomena spatio-temporal — posisi sendi saat ini hanya bermakna dalam konteks posisi sebelum dan sesudahnya. 3D-CNN memperlakukan sumbu waktu sebagai dimensi ketiga konvolusi, sehingga filter dapat mempelajari pola gerakan secara holistik.

"3D-CNN efektif menangkap fitur spasial (posisi sendi) dan temporal (waktu) secara bersamaan. Penggunaan Global Average Pooling 3D menekan jumlah parameter sehingga model dapat berjalan efisien secara real-time pada komputasi menengah." — Alsaedi et al. (2021); Chen et al. (2022)

Global Average Pooling 3D menggantikan fully-connected layer yang besar, mereduksi risiko overfitting dan mengurangi jumlah parameter secara signifikan — sangat cocok untuk dataset PoC berukuran kecil.

Landasan Teori
Domain-Informed + Decoupled Attention

Model deep learning murni bersifat black box — ia belajar korelasi statistik tanpa memahami anatomia tubuh. Atensi biomekanis menyuntikkan pengetahuan struktural (sendi mana yang penting untuk gerakan apa) langsung ke dalam proses komputasi model.

"Menggabungkan data-driven learning dengan pengetahuan struktur tubuh (domain-informed attention) membuat model lebih cerdas dan interpretatif layaknya pelatih profesional." — Hu et al. (2023)
"Pemisahan atensi spasial dan temporal (decoupled) meningkatkan fokus model pada sendi dan waktu yang relevan." — Xu, Shi & Zhang (2024)

# model_3dcnn.py — Fusi Atensi Biomekanis raw_attn = self.attention_layer(features) # Spatial Prior Mask: bobot per sendi per gerakan prior_mask = get_biomech_mask(exercise_type) fused_attn = raw_attn * prior_mask # Normalisasi Min-Max + Threshold norm_attn = (fused_attn - attn.min()) / (attn.max() - attn.min()) critical_joints = norm_attn[norm_attn > 0.4] # Sendi dengan nilai > 0.4 divisualisasikan merah (kritis)

Contoh: Untuk gerakan Squat, prior mask memberikan bobot tinggi pada pinggul dan lutut, sehingga hanya sendi tersebut yang "menyala merah" pada visualisasi atensi.

Landasan Teori
Dari mana label "benar/salah" berasal?

Supervised learning membutuhkan label. Untuk dataset latihan beban, pelabelan manual oleh pelatih bersertifikat mahal dan lambat. Validator biomekanis mengotomatisasi proses ini dengan meng-encode standar klinis dari literatur kedokteran olahraga ke dalam aturan matematika.

"Evaluasi kedalaman squat yang valid membutuhkan sudut lutut ≤ 100° dan pada deadlift diperlukan postur tulang belakang netral dengan inklinasi ≤ 60°." — Chen et al. (2022), Sensors
"Kesalahan pelebaran lutut ke dalam (Knee Valgus) dideteksi jika rasio lebar lutut terhadap pinggul < 0.85." — Rao et al. (2023)
"Rentang gerak (ROM) ekstensi siku yang tidak memadai pada Bench Press dideteksi dengan ambang batas sudut siku > 85°." — Ko et al. (2024), IEEE Access

# biomechanics_validator.py — Sudut Sendi 3D import numpy as np def joint_angle_3d(a, b, c): """Sudut ABC dengan arccosine vektor BA dan BC.""" ba = a - b; bc = c - b cos_angle = np.dot(ba, bc) / ( np.linalg.norm(ba) * np.linalg.norm(bc) + 1e-8) return np.degrees(np.arccos(np.clip(cos_angle, -1, 1))) # Contoh validasi Squat knee_angle = joint_angle_3d(hip, knee, ankle) label = 1 if knee_angle <= 100 else 0 # 1=benar

Aturan Threshold per Gerakan:

  • 🏋️
    Squat — Kedalaman Valid
    angle_knee ≤ 100° → label = 1 (BENAR)
  • 🔩
    Deadlift — Postur Tulang Belakang
    spine_inclination ≤ 60° → label = 1 (BENAR)
  • 🦵
    Knee Valgus — Pelebaran Lutut ke Dalam
    knee_hip_ratio < 0.85 → label = 0 (SALAH)
  • 🏆
    Bench Press — ROM Siku
    angle_elbow > 85° → label = 0 (TIDAK PENUH)

Hasil label ditulis otomatis ke kolom is_correct pada file dataset_manifest.csv, siap digunakan sebagai target supervised learning.
